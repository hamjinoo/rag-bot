# 상품 패키지 & 영업 자산 가이드

## 개요
외주 수주와 구독 비즈니스를 위한 **상품 구성, 가격 전략, 문서 템플릿**을 정리했습니다. 이 문서를 기반으로 노션 소개 페이지, 제안서, 계약서, 플랫폼 등록을 준비하세요.

---

## 1. 패키지 라인업

| 패키지 | 가격(원) | 포함 기능 | 납기 | 유지보수 |
| --- | --- | --- | --- | --- |
| Lite | 3,000,000 | RAG Q&A(문서 업로드), 텔레그램 또는 웹 1채널, 출처 표시, 기본 리포트 | 2주 | 2주 QA 지원 |
| Pro | 8,000,000 | Lite + 카카오톡, 관리자/로그/비용 모니터링, 정확도 리포트, 주 1회 미팅 | 4주 | 1개월 운영 지원 |
| Enterprise | 15,000,000~ | Pro + 권한/SSO, 온프레미스 배포, DB/Notion 연동, 맞춤 가드레일 | 협의 | 별도 계약 |

### 구성 문구 템플릿
- **Lite**: “사내 FAQ나 고객센터 FAQ를 빠르게 챗봇으로 전환하고 싶은 팀에 적합합니다. 2주 내 데모부터 납품까지 진행하며, 출처 표기를 통해 내부 검증 시간을 최소화합니다.”
- **Pro**: “두 개 이상 채널을 동시에 운영하고, 사용량/비용을 명확히 보고해야 하는 팀을 위한 플랜입니다. 관리자 페이지와 정확도 리포트를 포함해 품질 평가가 가능합니다.”
- **Enterprise**: “보안 요구가 높은 금융/대기업 고객을 위한 맞춤 구축 플랜입니다. 권한 체계, 온프레미스 배포, 외부 시스템 연동까지 포함합니다.”

### 추가 옵션
- 추가 채널(슬랙/팀즈): 1,200,000원
- 다국어 지원: 1,000,000원
- 사용자 교육(2시간): 500,000원
- 연간 유지보수: 월 과금의 15~20%

## 2. 구독형 요금제

| 요금제 | 월 비용(원) | 월 토큰 한도 | 리포트 | 지원 |
| --- | --- | --- | --- | --- |
| Starter | 190,000 | 20,000 토큰 | 이메일 | 3 영업일 이내 |
| Growth | 490,000 | 100,000 토큰 | 월간 리포트 | 24시간 이내 |
| Biz | 990,000+ | 맞춤 | 주간 리포트 + 개선 스프린트 | 전담 매니저 |

> 초반에는 외주(Lite/Pro)로 초기 자금 확보 → 구독(Starter/Growth)로 안정적 MRR 확보 전략을 권장합니다.

### 구독 고객 온보딩 체크리스트
- [ ] 고객 문서 수집(형식, 경로, 갱신 주기)
- [ ] 채널 선택(웹/텔레그램/카카오 등)
- [ ] 사용량 추적 지표 정의(정확도, 응답시간, 만족도)
- [ ] 월별 토큰 초과 시 과금 정책 안내
- [ ] 월간 리포트 샘플 공유 및 KPI 합의

## 3. 세일즈 프로세스
1. **리드 확보**
   - 숨고/크몽/위시켓, 링크드인, 기존 네트워크
   - 노션 랜딩 페이지 공유
2. **니즈 파악**
   - 문서 유형, 사용자 수, 보안 요구, 기대 정확도 질문
   - 예산 범위 확인 후 패키지 추천
3. **데모 제공**
   - 텔레그램/웹/카카오 데모 영상·스크린샷
   - 정확도 리포트 하이라이트
4. **제안서 전달**
   - 표준 제안 템플릿 기반 커스터마이징
   - 일정, 산출물, 유지보수, 비용 명시
5. **계약/착수**
   - 계약금(예: 50%) 선 청구, 잔금 조건 명시
   - 킥오프 미팅 → 프로젝트 관리 툴(노션/트렐로) 공유

### 미팅 스크립트 예시
1. **문제 파악 (5분)**  
   “현재 문서가 어떤 형식으로 정리돼 있고, 어떤 질문이 반복되고 있나요?”
2. **솔루션 설명 (10분)**  
   아키텍처 개요 + 텔레그램/웹 데모 바로 시연
3. **범위 정의 (10분)**  
   문서량, 사용자 수, 보안 요구(온프레미스 여부) 정리
4. **일정/비용 안내 (5분)**  
   적합한 패키지 추천, 옵션(권한/로그) 추가 여부
5. **다음 액션 (5분)**  
   제안서 발송 일정, 추가 자료 요청 사항, 다음 미팅 날짜 확정

## 4. 템플릿 체크리스트

### 4.1 노션 소개 페이지
- [ ] 문제/해결/데모/정확도/보안/가격/FAQ/연락처 섹션
- [ ] Demo 영상 3개 (텔레그램/웹/카카오)
- [ ] 정확도 지표(Top-k, MMR, 임계치) 시각화
- [ ] 보안 정책 (권한, 로그, 개인정보 미수집)
- [ ] CTA 버튼 (미팅 예약/이메일)

### 4.2 제안서 (PPT 또는 노션)
- 표지: 고객명, 프로젝트명, 작성일
- 배경/문제 정의
- 솔루션 개요(아키텍처, 기술 스택)
- 산출물 목록 및 납기 일정
- 비용/결제 조건
- 유지보수/운영 지원 범위
- 팀 소개 및 레퍼런스

### 4.3 견적서
- 기본 포맷: 공급가, 부가세, 총액
- 옵션 항목 별도 라인 분리
- 유효 기간(예: 14일) 명시

### 4.4 계약서
- 범위 정의(Scope of Work)
- 산출물 및 검수 기준
- 지불 조건(착수금/중도금/잔금)
- 비밀유지조항(NDA)
- 지적재산권 귀속
- 하자보수 기간 및 책임

## 5. 마케팅 자산
- **데모 영상**: 30~90초, 무음 + 자막, 핵심 기능 강조
- **블로그/링크드인 글**: 구축 과정, 정확도 개선 사례 공유
- **FAQ 스크립트**
  - Q: “AI 헛소리 방지 어떻게 하나요?” → 가드레일 + 임계치 설명
  - Q: “보안 문제 없나요?” → 온프레미스, 권한, 로그 정책 소개
  - Q: “데이터 갱신은?” → 업로드 파이프라인/자동화 옵션 안내
- **후속 Follow-up 메일 템플릿**
  ```
  안녕하세요, 지난 미팅 이후 논의드린 RAG 챗봇 제안서 공유드립니다.
  첨부된 문서를 확인해 주시고, 궁금한 점 있으시면 편하게 회신 부탁드립니다.
  이번 주 안에 진행 여부만 알려주시면 일정 확보에 큰 도움이 됩니다.
  ```

## 6. 운영 KPI
- 리드 → 제안서 전환율
- 제안서 → 계약 전환율
- 구축 기간 평균(주)
- 정확도(정답률, 근거 일치율)
- 고객 만족도 (CSAT, NPS)
- 월간 재방문/질문 수 (운영 안정성 지표)
